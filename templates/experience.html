EDS App Backend Robust and Scalable Architecture for Service Stations

The backend of EDS App is built with ASP.NET Core, following a hexagonal architecture that ensures clean, decoupled, and maintainable code. It implements JWT for secure token-based authentication and exposes REST APIs documented with Swagger.

The solution is optimized for production environments using Redis for data caching, Docker for containerization and portable deployment, and AWS as the cloud platform. It also uses GitHub Actions to automate CI/CD pipelines. All development and orchestration are managed through Visual Studio, providing high productivity and full control over the development lifecycle.

Key Technologies

    Core Framework: ASP.NET Core
    Security: JWT token-based authentication
    API Documentation: Swagger
    Performance: Redis for caching
    Containers & Deployment: Docker, AWS
    CI/CD Automation: GitHub Actions
    Development Environment: Visual Studio
    Architecture: Hexagonal – clear separation between domain, infrastructure, and adapters

Role in the Project

Actively contributed to the full development lifecycle of the project as part of a collaborative development team. Responsibilities included designing the architecture, implementing mobile and backend services, setting up automated testing and CI/CD pipelines, and ensuring cloud readiness and security best practices.

EDS App is a modern application developed with .NET MAUI, designed to manage and operate service stations.
